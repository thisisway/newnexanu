# 03 — Design System

## Personalidade visual

O Nexano deve transmitir:

- Confiança
- Clareza
- Modernidade
- Segurança
- Velocidade
- Controle
- Simplicidade
- Profissionalismo

A interface deve ser premium, minimalista e altamente funcional.

## Referências conceituais

Usar como inspiração de qualidade, sem copiar:

- Linear pela clareza
- Stripe pelo billing e organização
- Vercel pela simplicidade técnica
- Railway/Render pela experiência cloud
- Intercom pelo suporte
- Notion pela suavidade

## Cards

Cards devem ser o principal elemento visual da interface.

Um card deve conter:

- Título claro
- Subtítulo opcional
- Status quando necessário
- Métrica ou conteúdo principal
- Ação principal
- Ação secundária quando útil
- Estado visual coerente

## Estados vazios

Estados vazios devem ensinar e orientar.

Exemplo para Sites:

> Você ainda não tem sites.  
> Vamos criar seu primeiro site em poucos minutos.

Ações:

- Criar WordPress
- Migrar site
- Publicar app
- Falar com suporte

## Microinterações

- Hover suave em cards
- Loading skeleton
- Toasts claros
- Botão copiar com confirmação
- Transições suaves em steppers
- Atualização em tempo real de status
- Confirmações antes de ações destrutivas

As animações devem ser discretas, rápidas e profissionais.
